    How It Works

    그러면, 어떻게 만드나?

    AI에게 말로 설명하면 직접 만들어줍니다. 단, 처음 요청이 중요합니다.

    📋 처음 설계가 핵심!

    중간에 바꾸면 처음부터 다시 만드는 것과 같습니다

    ❌ 나쁜 요청
    "출퇴근앱 만들어줘"
    → "여기 바꿔"
    → "저것도 추가"
    → 결국 처음부터 다시
    ✅ PRD 방식 요청
    1. 목적: 출퇴근 기록 웹앱
    2. 기능: 출퇴근 버튼, 월별 조회
    3. 화면: 메인, 기록 목록
    4. 저장: 브라우저 로컬

    🎨 예시: v0에서 첫 요청

    말로 설명하면 화면과 코드를 한 번에 만들어줍니다

    Prompt Tips

    만든 다음에는, 이렇게 수정합니다

    처음 만든 결과물을 다듬을 때도 구체적으로 말할수록 결과가 좋습니다.

    ❌ 나쁜 수정 요청 ✅ 좋은 수정 요청
    "좀 바꿔줘" "헤더 배경을 파란색으로, 글씨는 흰색으로 바꿔줘"
    "에러 고쳐줘" "버튼 누르면 화면이 안 바뀌어. 콘솔에 에러 메시지 복사해줄게"
    "더 예쁘게" "카드에 그림자 넣고, 카드 사이 간격 좀 넓혀줘"
    🎯 한 번에 다 바꾸지 말고, 단계별로 요청하세요
    1. 확인: "지금 결과물 괜찮아. 여기서 수정할게"
    2. 하나씩: "테이블에 날짜 컬럼 추가해줘"
    3. 확인 후 다음: "좋아! 이번엔 차트도 넣어줘"
    4. 마무리: "모바일에서도 잘 보이게 정리해줘"

    앱만으로도 충분히 유용하지만, 여기에 AI를 연결하면 한 단계 더 나아갑니다.

    만든 앱 AI 없이도 이만큼은 됩니다 여기에 AI를 연결하면?
    📊 현황 대시보드 수동으로 데이터 입력하고, 차트로 표시 AI가 데이터 분석하여 이상 탐지·추세 예측까지 자동
    🔎 현장 점검 앱 체크리스트 입력하고, 사진 첨부해서 제출 AI가 사진 분석하고 불량·위험 자동 판별·리포트 생성
    📅 예약 / 신청 시스템 신청서 올리고, 관리자가 수동 승인 AI가 일정·조건 분석해서 자동 배정·승인 판단
    앱에 데이터를 넣으면 → AI가 읽고 판단 → 결과까지 자동으로
    MISO

    AI 자동화 흐름 만들기

    MISO의 플레이메이커 기능으로, AI 동작 흐름을 설명하면 AI가 만들어줍니다.

    1

    대화로 설명

    원하는 흐름을 말로 설명하면
    AI가 질문하며 함께 완성합니다

    2

    플레이메이커가 생성

    AI가 자동화 흐름을
    만들어줍니다

    3

    저장 → 발행

    "저장하기" 클릭 후
    "발행하기" 클릭

    4

    API 키 생성

    "앱 공유하기"
    → "다른 서비스와 연결"
    → API 키 생성
    → 키와 URL 복사

    5

    내 앱에 연결

    API 키와 URL을 앱에 넣으면 끝
    연결은 AI에게 맡기면 됩니다

    AI와 대화하며 흐름 설계 → 플레이메이커가 흐름 자동 생성 → API 키만 앱에 넣으면 연결 완료

    API란? — 서로 다른 프로그램끼리 데이터를 주고받는 통로. 내 앱에서 MISO에 데이터를 보내면, AI 결과를 돌려줍니다.

    Part 1

    환경 준비

    설치 없이 웹 브라우저만 있으면 됩니다 — 가입만 하면 준비 끝!

    Step 1: GitHub 가입

    1. github.com 접속
    2. "Sign up" → Google 계정으로 가입
    3. v0, Supabase 모두 GitHub 계정으로 로그인

    Step 2: v0 접속

    1. v0.dev 접속
    2. GitHub 계정으로 로그인
    3. AI에게 말하면 앱을 만들어주는 도구

    Step 3: Supabase 가입

    1. supabase.com 접속
    2. "Start your project" → GitHub 계정으로 가입
    3. 게임 기록 등 데이터 저장에 사용
    Part 2

    🎮 웹 게임 만들기

    Step 1: PRD 만들기

    먼저 GPT나 Gemini에게 게임 기획서를 만들어달라고 요청

    > v0에서 웹 게임을 만들려고 해.
    1984 스타일 세로 스크롤 비행기 슈팅 게임이야.
    적기가 위에서 내려오고 총알로 쏴서 점수를 올리는 거야.
    PRD를 마크다운으로 정리해서 작성해줘.

    Step 2: PRD 확인 후 게임 만들기

    PRD 확인 후 v0에게 게임 생성 요청

    > 좋아, 이 PRD대로 게임을 만들어줘.
    모든 UI는 한글로 만들어줘.

    Step 3: 결과 확인 & 플레이

    1. v0가 게임을 자동 생성
    2. 미리보기에서 직접 플레이!
    3. 재미있게 작동하는지 확인

    Step 4: 업그레이드 요청

    > Supabase에 연결해서 플레이어 이름 입력,
    점수 기록, 시도 횟수 표시 기능 추가해줘.
    모든 UI는 한글로 만들어줘.
    -- v0가 Supabase 연결 화면을 띄워줍니다
    -- Free Plan 선택 → Continue 클릭!
    Part 3

    💾 Supabase 확인 & 랭킹 보드

    Part 2에서 v0가 Supabase를 자동 연결했습니다 — 이제 확인하고 랭킹을 만들어봅시다

    Step 1: Supabase 대시보드 확인

    v0가 자동 생성한 프로젝트와 테이블 확인

    1. supabase.com → 로그인
    2. 자동 생성된 프로젝트 확인
    3. Table Editor에서 테이블 확인

    Step 2: 저장 테스트

    게임을 플레이하고 데이터가 저장되는지 확인

    1. v0 미리보기에서 게임 플레이
    2. 이름 입력 → 점수 기록
    3. Supabase Table Editor에서
      데이터가 들어왔는지 확인

    Step 3: 랭킹 보드 만들기

    랭킹 보드가 없다면 추가 요청

    > Supabase에 목업 데이터를 먼저 넣고,
    점수 높은 순으로 가져와서
    랭킹 보드를 보여줘.
    상위 10명까지 표시해줘.
    모든 UI는 한글로 만들어줘.

    🤖 한 걸음 더 — MISO AI 연결하기

    게임 플레이 데이터를 MISO에 보내서 AI 피드백을 받아보자

    Step 1: MISO에서 흐름 만들기

    miso.run → 새 흐름 생성 → 아래 프롬프트 설정 → 발행하기 → API Key, URL 복사
    ⚠️ MAX 토큰 넉넉하게 설정 / 발행하기 필수!

    -- MISO 시스템 프롬프트 (흐름에 붙여넣기)
    너는 슈팅 게임 "SKY FORCE 1984"의 AI 코치야.
    플레이어의 게임 기록을 분석해서 피드백을 줘.

    응답 규칙: 한국어, 존댓말, 이모지 사용, 300자 이내

    응답 형식:
    🎖️ 이번 비행 평가 (한 줄 총평)
    ✅ 잘한 점 (2~3개)
    💡 다음 비행 팁 (2~3개)
    🔥 도전 목표 (1개)

    Step 2: v0에 한번에 붙여넣기

    아래 내용을 통째로 복사해서 v0에 붙여넣기
    붙여넣기 후 v0 Settings → Environment Variables에 MISO_API_KEY, MISO_API_URL 추가

    게임 중 플레이어 행동(이동, 발사, 처치, 피격)을 임시 기록하고,
    게임 종료 시 MISO API로 보내서 AI 코치 피드백을 결과 화면에 보여줘.
    모든 UI는 한글로 만들어줘.

    MISO API 연결 참조:
    - Endpoint: POST {MISO_API_URL}/chat
    - Headers:
    Authorization: Bearer {MISO_API_KEY}
    Content-Type: application/json
    - Body:
    {
    "inputs": {},
    "query": "플레이 기록 텍스트",
    "mode": "blocking",
    "conversation_id": "",
    "user": "player"
    }
    - 응답에서 data.answer가 AI 피드백 텍스트

    환경 변수 (서버 사이드에서 사용):
    - MISO_API_KEY=발행 후 복사한 키
    - MISO_API_URL=https://api.holdings.miso.gs/ext/v1

    수고하셨습니다.
